## Axis 1: Tight-knit invite-only / minimalist (4)

### Lobste.rs — https://lobste.rs *(fetched 2026-05-17)*

**IA:** Active / Recent / Comments / Search / Login top-nav. Reverse-chronological ranked list. Each item: vote count + tags + title + source domain + submitter avatar + comment count + Archive.org/Ghostarchive links.

**Visual:** Monochromatic, text-dense, sparse imagery (only small avatars), blue hyperlinks. Zero hero/carousel/decoration.

🔑 **Borrow:**
- *Tag-first categorization* (vs hierarchical sub-forums)
- *Visible member attribution* on every submission (submitter name + avatar as first-class metadata)
- *Archival transparency* (every link backed up via Archive.org/Ghostarchive)
- *Invite-tree visible per user* — unique social-proof primitive ("invited by @anton, who was invited by @founder")

### Hacker News — https://news.ycombinator.com *(fetched 2026-05-17)*

**IA:** new / past / comments / ask / show / jobs / submit. Pure points-ranked list. Story = headline + domain + points + submitter + timestamp + comments. No images, no ads, no decoration.

🔑 **Borrow:**
- *Extreme minimalism* — reduces cognitive load, page weight
- *Transparent ranking* (points + comments visible to everyone)
- *Consistent template* across all surfaces — same row shape on /new, /past, /comments
- *Direct source attribution* (domain in parens next to headline)

Durability: 20 years, design barely changed. The opposite of engagement-maximizing.

### Lex.page (status: training-data only — invite-only AI writing community pre-2025; current status not verified)

**Pattern (historical):** invitation-only AI-assisted writing community; documents as social objects; soft community via comments + follows on docs.

**Borrow:** *invitation as commitment device* — friction at the door produces engagement on the inside (consistent across Lobsters, Lex, pre-shutdown Read.cv).

### Read.cv — ⚠️ DEAD (acquired by Perplexity Jan 2025, shut down May 2025)

**Pattern reference only.** Site no longer operational.

**IA pattern (historical):** profile-first; CV-as-canvas; subtle community signals (follows, comments, work updates). Designer-grade aesthetic with generous whitespace.

🔑 **Borrow:** *profile aesthetic as content* — member pages were the platform's centerpiece, not an afterthought. Showcase work + presence + work updates. Lesson: the platform died for business reasons, not design — the pattern is reusable.

## Axis 2: Modern community SaaS (3)

### Circle.so — https://circle.so *(fetched 2026-05-17)*

**IA:** integrated dashboard — Community + Messaging + Courses + Events + Payments + Email + Website Builder + AI agents. Personalized algorithmic feed. Native iOS/Android apps.

**Visual:** clean professional minimalism, gradients, spacious layouts, creator testimonials (Jay Shetty, Tim Ferriss).

**Borrow:** *unified module surface* (community + events + payments coexist); *personalized feed*; *native mobile-first*.

⚠️ **AVOID for v0.4:** AI agents / built-in monetization / SaaS-feature-creep is the wrong scope. The "platform-as-product" framing doesn't fit a member-only Warsaw community.

### Geneva (training-data)

**IA:** club-based; channels + events + audio rooms inside each club. Mobile-first.

**Borrow:** *feature-rich without overwhelm* via clear hierarchy (club → channel → message).

### Mighty Networks (training-data)

**IA:** community + courses + events + cohorts combined.

⚠️ **AVOID:** course-heavy framing inappropriate for a meetup-based community.

## Axis 3: Dev/tech community-org (4)

### PostHog Community — https://posthog.com/community *(fetched 2026-05-17)*

**IA:** community-newspaper format. Editor's note + hedgehog wisdom → team member spotlight → people news → newsletter signup → changelog → merch → CEO musings → feature request voting.

**Visual:** branded voice (hedgehog mascot), custom illustrations, playful but professional.

🔑 **Borrow:**
- *Human-centric design* — team spotlights, people news = community IS people
- *Radical transparency* — changelog + roadmap + handbook public-by-default
- *Branded voice* — personality, not sterile corporate
- *Curated over chronological* — editor selects, not engagement-maximizing feed

### Linear — https://linear.app *(fetched 2026-05-17)*

**IA:** Product / Resources / Customers / Pricing / Docs top nav. Footer reorganizes by Product / Features / Company / Resources / Connect / Legal.

**Visual:** monochromatic palette (deep blacks/whites/grays), sans-serif typography, generous whitespace, functional imagery (interface screenshots), muted photography.

🔑 **Borrow:**
- *Whitespace as structure* — breathing room defines sections, not borders
- *Monochromatic + accent* — black/white foundation + selective color for CTAs
- *Typography-first hierarchy* — scale and weight do the work
- *Functional over decorative imagery* — screenshots, data viz, authentic context
- *Progressive disclosure* — hide complexity, reveal on demand

Premium-without-shouting aesthetic. Tightest reference for "minimal but feels expensive."

### Vercel Community (training-data — vercel.com/community)

**IA:** docs + showcase + integrations + community combined; consistent header across the entire vercel.com brand surface.

**Borrow:** *cross-linking between content types* — a project showcase links to docs links to community discussion. No content silos.

### dev.to + IndieHackers (training-data)

- **dev.to:** tag-driven feed; author profile prominent. Borrow: *author as content equal-partner*.
- **IndieHackers:** stories + groups + products; revenue-as-content. Borrow: *builder-showcase as social currency*.

⚠️ **dev.to avoid:** cluttered feed with competing CTAs and dense content.

## Axis 4: Event-forward (2)

### Luma — https://luma.com *(fetched 2026-05-17)*

**IA:** Discover / Pricing / App / Help. Main verbs: "Create Your First Event" + "Discover Events." Discovery hub at /discover.

**Visual:** "Delightful events start here." — outcome-focused messaging, clean & professional.

🔑 **Borrow:**
- *Dual-path CTA* — creator vs attendee paths split at the door (not buried in a generic "Sign up")
- *Event detail page as platform spine* — the page IS the product
- *Value proposition before features* — outcome messaging not feature lists

### Meetup (training-data)

**IA:** groups + events + RSVPs. Older UX but proven flow.

**Borrow:** *recurring meeting cadence visibility* — the weekly rhythm becomes part of the IA.

## Cross-cutting traits (🔑 appeared in 3+ best-in-class)

1. **Brand restraint** — Linear, Lobsters, HN: minimal palette + typography-first + whitespace as structure. Nothing screams.
2. **Member-as-content** — Read.cv, PostHog (team spotlights), dev.to (author profile): profiles equal-partner, not afterthought.
3. **Events as platform spine** — Luma: events get beautiful detail pages with clear CTAs.
4. **Transparent ranking / archive** — HN, Lobsters: points + comments + archival links visible to all.
5. **Human-centric over feature-centric** — PostHog newspaper, Read.cv profiles: people show through, not SaaS chrome.
6. **Curated > chronological** — PostHog editor's selects, HN ranking: human curation or transparent algorithm, never engagement-maximizing.

## Anti-patterns to avoid (⚠️)

1. **Circle.so SaaS-feature creep** — payments / courses / AI moderation are wrong scope for v0.4.
2. **dev.to cluttered feed** — competing CTAs, dense content, no breathing room.
3. **Mighty Networks course-heavy framing** — doesn't fit a meetup-based community.
4. **Generic forum aesthetic** — sterile, all-thread, no personality.

---

## Closest matches for Warsaw AI Community v0.4

**By character:** Lobste.rs aesthetic (minimalism + tag-first + visible member reputation) + Read.cv member profiles (designer-grade showcase) + Luma event UX (beautiful detail pages + dual-path) + PostHog transparency (handbook + roadmap public + branded voice).

**Closest single reference:** Linear, for restraint and confidence; PostHog, for human-centric warmth.

**Worst fits:** Circle.so (too SaaS), Mighty Networks (too coursey), Geneva (too app-store), dev.to (too cluttered).
