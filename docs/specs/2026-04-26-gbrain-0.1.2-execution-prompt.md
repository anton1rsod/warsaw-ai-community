## Why this prompt is shaped the way it is

- **Reads four files in order** (plan first, spec second, ADRs third, closeout fourth) — same pattern as `2026-04-26-gbrain-next-chat-prompt.md` that successfully kicked off this brainstorm chat. Plan-first is correct because the plan is the action document; the spec is reference.
- **Names the sub-skill** explicitly (`superpowers:subagent-driven-development` for this size of plan; `superpowers:executing-plans` is the inline alternative). The plan's header already references both, but stating it here saves one round-trip.
- **Constraints section repeats the Phase-E firewall from this chat's prompt** — keeps the execution chat from accidentally touching real-channel infrastructure. This is the most important guardrail.
- **Anton's preferences** are repeated verbatim from the brainstorm prompt — execution may need to surface decisions where the bot would otherwise present a neutral menu.
- **Three human-in-the-loop steps** are surfaced upfront so the executor doesn't ask Anton to fix STAGING_CHAT_IDS via bot tools (e.g., trying to grep his Telegram).
- **Acknowledgment-first** matches how this brainstorm chat opened (acknowledge closeout, then proceed) — gives Anton a confirm-or-redirect handle before code starts moving.

## What's deliberately NOT in the prompt

- Detailed task-by-task instructions — those live in the plan; duplicating here would rot.
- The three ADRs' contents — they're skim-only context. The plan references them where relevant.
- The questionnaire — irrelevant to execution; brainstorming questions are settled.
- The v1 superseded spec — confusing; v2.1 is canonical.
- Any mention of `/summarize` work — explicitly out of scope.
