## Don't repeat (lessons from chat 14)

1. **Vercel Sensitive env vars are write-only** — `vercel env pull` silently omits `CRON_SECRET`, `GEMINI_API_KEY`, etc. Don't propose curl-with-secret diagnostics; use Cron Jobs "Trigger Now" UI button (Pro plan only) or wait for natural fire.
2. **Don't paste API keys in chat.** If a key lands in the transcript, flag and force-rotate immediately. Anton's `feedback_secret_handling` memory rule.
3. **Two repos under one user is a footgun** — repo migrations can orphan Vercel Git connections silently. Always check the Vercel project's Git settings after any repo rename/migration.
4. **CWD drifts in Bash sessions** — earlier `cd projects/gbrain/app` left the shell there for the rest of the session. Use absolute paths or re-cd to repo root explicitly.
5. **0.1.2's `commands/ask.ts` calibration note** at line 13–20 is the new source-of-truth comment for the threshold rationale. Don't strip it on future edits — point future calibration commits at it.
