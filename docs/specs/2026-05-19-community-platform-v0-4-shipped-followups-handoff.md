### Option A — PWA icon textured "WA" upgrade

**Cost:** ~30 min. Local-only — Anton runs ImageMagick on his machine.

**Scope:** Replace the solid `#f59e0b` placeholder icons (currently `icon-192.png` / `icon-512.png` / `apple-touch-icon.png` / `favicon.ico` generated by `scripts/regen-pwa-icons.ts`) with textured "WA Inter Semibold white on amber" per Q4.7 / D34. The plan's Path A in v0.4.0-plan.md A.3.1 is the canonical recipe (ImageMagick `magick` invocations).

**Why deferred:** The v0.4.0 chat-25 sandbox lacked ImageMagick + `sharp`; Anton's local environment likely has one of them.

**Acceptance:** PNG files render "WA" centered on amber at correct dimensions; `file public/icons/icon-192.png` → "PNG image data, 192 x 192, 8-bit/color RGBA"; visual check in browser DevTools manifest panel.

**Commit:** `feat(community-platform): textured WA icons (v0.4.1 / Q4.7 D34 follow-up)`. PR if touching CI; direct-to-main if assets-only per `feedback_pr_vs_direct` memory.

### Option B — YourWeekPane data-source wiring

**Cost:** ~1.5 hours. New code + tests.

**Scope:** Replace the stub helpers in `app/home/page.tsx`:
- `nextRsvpForMember(handle)` currently returns null → wire to `lib/__generated__/event-rosters.json` aggregated with `listEventsFromSnapshot()`.
- `kudosThisWeekFor(handle)` currently returns 0 → wire to `lib/kudos.ts` (existing v0.3 helper).

Extract into `lib/your-week.ts` for testability. Add unit tests asserting:
- next-RSVP returns the soonest upcoming event the member RSVP'd "Going" to.
- next-RSVP returns null when no future RSVPs exist.
- kudos-this-week counts the current-ISO-week-window kudos received by the member.

**Why deferred:** Phase A's YourWeekPane shipped with stubs to keep the chat-25 scope tight; the component's empty-state copy + i18n string surface is correct. Data-source wiring is mechanical.

**Acceptance:** Signed-in `/home` shows next-RSVP commitment when one exists + kudos count when > 0; empty-state copy when no commitments. New `lib/your-week.test.ts` covers all branches.

**Commit:** `feat(community-platform): YourWeekPane data sources (v0.4.1 / O9 wire-up)`.

### Option C — HomeFeed `relativeDate()` → `<DateTime>` migration

**Cost:** ~30 min.

**Scope:** `app/components/HomeFeed.tsx` still ships v0.3's inline `relativeDate()` helper. Phase A introduced `<DateTime context="list">` which does the same thing via `lib/i18n/strings.ts` (H67-compliant). Migrate HomeFeed to consume `<DateTime>` instead.

**Why deferred:** A.2.4's plan-text suggested this migration but the chat-25 implementer kept HomeFeed structurally unchanged to minimize regression surface during the YourWeekPane + HomeHeader-delete dance.

**Acceptance:** HomeFeed.tsx no longer defines `relativeDate()`; all date-fragment outputs consume `<DateTime context="list">`; existing HomeFeed tests pass (or are updated to assert against DateTime's output format).

**Commit:** `refactor(community-platform): HomeFeed consumes <DateTime> (v0.4.1 / H67 H67-adjacent)`.
